Overview

Visão Geral

Uma extensão Chrome (Manifest V3) para detectar e gerenciar abas duplicadas de forma segura e controlada, com arquitetura modular e preparada para evolução.

🔍

O que faz

Executa um scan das abas abertas, normaliza URLs e agrupa duplicatas para análise e ações.

🛡

Segurança & Privacidade

Processamento local no navegador: sem envio de dados para servidor. Ações de fechamento/movimento são sempre iniciadas pelo usuário.

🧩

Arquitetura

Camadas claras: core (lógica), browser (Chrome APIs), orchestration (fluxos) e popup (UI).

⚙

Principais ações

Fechar duplicatas mantendo 1, mover para nova janela e agrupar abas duplicadas (quando suportado).

🚀

Como usar (30s)

Abra o popup → clique em Scan Tabs → revise grupos → execute a ação desejada com feedback na UI.

Fluxo de alto nível ›
1
Coletar
Busca abas via Chrome API e filtra o que não deve entrar no scan.
2
Normalizar
Remove ruídos (ex.: hash/UTM) para comparar URLs de forma consistente.
3
Agrupar
Identifica duplicatas e calcula estatísticas para orientar ações.
4
Agir
Usuário executa fechar/mover/agrupar e a UI atualiza o resultado.

Quick Start

Instalação + Uso
1Garanta que a estrutura do projeto esteja correta.
2Abra chrome://extensions/ e ative o modo desenvolvedor.
3Carregue a pasta com o manifest.json.
Passo a passo ›
  1. Verifique se a pasta raiz contém manifest.json e as pastas core/, browser/, orchestration/ e popup/.
  2. Abra chrome://extensions/ e ative Modo do desenvolvedor.
  3. Clique em Carregar sem compactação e selecione a pasta do projeto.
  4. Fixe o ícone: Quebra-cabeça → “Tab Duplicate Detector” → alfinete.
  5. Abra o popup e clique em Scan Tabs para gerar a lista de duplicatas e estatísticas.
Checklist de validação ›
  • A extensão aparece em chrome://extensions sem erros.
  • O ícone fica visível na barra (fixado no alfinete).
  • Ao clicar em Scan Tabs, o popup exibe estatísticas e/ou grupos.
Testes rápidos sugeridos ›
  • Normalização: URLs como #readme e utm_source devem cair no mesmo grupo.
  • Ações: “Fechar duplicatas” fecha as demais e mantém uma aba aberta (reversível via histórico do navegador).
  • Mover/Agrupar: “Mover para nova janela” cria nova janela; “Agrupar abas” usa Tab Groups.

Guia prático do que a extensão faz e como usar.

S

Varredura de Abas

Analisa todas as abas abertas em todas as janelas e identifica duplicatas por URL normalizada.

Desenvolvimento

Como evoluir
⏱

Visão Geral

Decisões práticas para manter o projeto simples, seguro e evolutivo.

A extensão é construída em JavaScript com ES6 Modules e Manifest V3, sem dependências externas. A separação por camadas mantém a lógica de negócio independente das APIs do Chrome, facilitando manutenção e futuras melhorias.

🛡

Segurança

Permissões mínimas e ações explícitas do usuário.

A extensão trabalha localmente com abas do navegador; não envia dados para rede. As operações que alteram estado (fechar/mover/agrupar) só ocorrem quando o usuário confirma no popup.

Destaque: validações antes de executar ações em orchestration/action-orchestrator.js.
⚡

Desempenho

Scan rápido e renderização incremental.

O scan coleta dados essenciais e agrupa por URL normalizada em memória. A UI renderiza estatísticas e grupos de forma direta, evitando processamento pesado no popup.

Destaque: normalização reduz duplicidade falsa em core/url-normalizer.js.
▦

Estrutura & Boas Práticas

Camadas com dependências unidirecionais.

core/ não depende do Chrome; browser/ isola APIs; orchestration/ coordena fluxos; popup/ cuida da UI.

Core Browser Orchestration UI
Destaque: lógica de duplicatas fica em core/duplicate-detector.js (puro).
🧪

Testes & CI

Prioridade para estabilidade do fluxo principal.

Não há suíte automatizada no runtime atual. O caminho natural é começar por testes de unidade do core/ (normalização e detecção).

Planejado: adicionar testes do core e checagens automáticas em pipeline (CI).
👥

Contribuição

Como adicionar regras, ações e UI sem quebrar camadas.

Para evoluir com baixo risco, mantenha mudanças de regra no core/. Ações devem ser implementadas em browser/tabs-actions.js e orquestradas em orchestration/action-orchestrator.js. A UI fica concentrada em popup/popup-controller.js e popup/ui-renderer.js. 

Onde mexer:
  Nova regra: core/url-normalizer.js
  Nova ação: browser/tabs-actions.js + orchestration/action-orchestrator.js
  Novo fluxo de scan: orchestration/scan-orchestrator.js
  UI/Interações: popup/popup-controller.js + popup/ui-renderer.js
Destaque: manter core puro evita acoplamento com Chrome API.

Estado e erros ›
  • Sem estado global: módulos core e adapters são desenhados de forma stateless/pura.
  • Estado local: popup-controller.js mantém estado do último scan em memória.
  • Estratégia de erro por camada: Browser pode falhar com contexto; orquestração captura e transforma em objetos de resultado; UI exibe mensagens amigáveis.
