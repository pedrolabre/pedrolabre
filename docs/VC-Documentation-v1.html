Uma solução moderna e modular construída com Node.js, Express e SQLite.

Tecnologias

Stack moderna, modular e escalável com as melhores práticas

O Armário Virtual foi construído com tecnologias modernas e confiáveis, seguindo princípios de Clean Code, Modularização e Separação de Responsabilidades. Cada arquivo possui menos de 150 linhas, facilitando manutenção e testes.

Backend
Node.js 18+ JavaScript Runtime

Uso no Projeto: Runtime JavaScript server-side que executa toda a lógica do backend. Permite usar JavaScript tanto no frontend quanto no backend, facilitando o desenvolvimento full-stack.

Implementação: Servidor HTTP configurado no server/index.js que escuta na porta 3000. Gerencia requisições, rotas, middlewares e comunicação com o banco de dados SQLite.

Express 4.x Framework Web

Uso no Projeto: Framework web minimalista para Node.js que simplifica a criação de APIs REST. Gerencia rotas, middlewares e respostas HTTP.

Implementação: Configurado para servir arquivos estáticos da pasta public/, processar JSON no body das requisições, e organizar rotas em server/routes/ com controllers separados por funcionalidade (items, looks, categories, etc.).

SQLite Banco de Dados

Uso no Projeto: Banco de dados relacional leve e sem servidor, perfeito para aplicações locais. Armazena todas as peças de roupa, looks, categorias, notas e estatísticas.

Implementação: Arquivo wardrobe.db em data/ com tabelas relacionadas. Queries SQL organizadas em módulos (server/database/items.js, looks.js, etc.) seguindo o padrão Repository. Suporta transações e relacionamentos entre tabelas.

Multer Upload de Arquivos

Uso no Projeto: Middleware para Node.js que processa uploads de arquivos multipart/form-data. Permite que usuários façam upload de até 5 fotos por peça de roupa.

Implementação: Configurado em itemController.js com .array('images', 5) para aceitar múltiplos arquivos. Salva imagens na pasta uploads/ com nomes únicos gerados por timestamp. Valida tipos de arquivo (JPEG, PNG) e tamanho máximo.

Frontend
HTML5 Estrutura

Uso no Projeto: Linguagem de marcação semântica que estrutura toda a interface do usuário. Usa tags modernas como <section>, <article>, <nav> para melhor acessibilidade.

Implementação: Arquivo principal public/index.html com estrutura modular: navegação, sections para cada funcionalidade (items, looks, calendar, statistics), modais para CRUD, e formulários com validação nativa. Atributos data-* para interação com JavaScript.

CSS3 Modules Estilização

Uso no Projeto: Estilização modular com 24 arquivos CSS separados por responsabilidade. Usa CSS Variables para temas, Flexbox e Grid para layouts responsivos, e transições para animações suaves.

Implementação: Arquivo principal main.css importa todos os módulos (base.css, layout.css, modal.css, etc.). Sistema de temas com :root e [data-theme="dark"]. Media queries em responsive.css para mobile, tablet e desktop.

JavaScript ES6+ 32 Módulos

Uso no Projeto: 32 módulos ES6 com responsabilidade única. Usa async/await para requisições, destructuring para dados, arrow functions e imports/exports nativos. Padrões Observer e Module para organização.

Implementação: Entry point app.js inicializa módulos. state.js gerencia estado global, api.js centraliza requisições fetch, módulos CRUD (items-crud.js, looks-crud.js) seguem padrão consistente. Event listeners em event-listeners.js, manipulação DOM em dom.js.

Font Awesome 6 Ícones

Uso no Projeto: Biblioteca de ícones vetoriais que fornece interface visual consistente e profissional. Mais de 100 ícones diferentes usados na navegação, botões, cards e feedback visual.

Implementação: CDN carregado no HTML via link <link rel="stylesheet" href="cdnjs...font-awesome">. Classes fas, far, fab aplicadas em elementos <i>. Ícones para categorias, ações (edit, delete, save), status e navegação.

APIs & Integrações
Open-Meteo API Previsão do Tempo

Uso no Projeto: API gratuita de previsão do tempo que fornece temperatura e condições climáticas atuais. Usado para sugerir roupas adequadas baseadas no clima (inverno, meia-estação, verão).

Implementação: Serviço weatherService.js faz requisições HTTP para a API com latitude/longitude. Controller weatherController.js processa resposta e filtra peças por atributo "estação". Cache de 30 minutos para evitar requisições excessivas. Fallback para modo offline.

LocalStorage API Armazenamento Local

Uso no Projeto: API nativa do browser para armazenamento local persistente. Salva preferências do usuário como tema (claro/escuro), cor do tema, e configurações de exibição.

Implementação: Módulo dark-mode.js salva/carrega tema com localStorage.setItem/getItem('theme'). Módulo color-theme.js persiste esquema de cores escolhido. Estado carregado no início da aplicação em app.js para aplicar preferências antes do render.

Problemas & Soluções

Desafios encontrados durante o desenvolvimento e como foram resolvidos

Botões de Navegação Não Funcionavam

Frontend
Resolvido

Problema

Os botões da navegação principal (Itens, Looks, Calendário, etc.) não respondiam aos cliques. O JavaScript estava sendo carregado mas os event listeners não eram anexados.

Causa

O arquivo app.js original era muito extenso, tornando difícil identificar onde os handlers eram registrados. Além disso, havia conflitos de escopo entre funções.

Solução

Refatoração completa do código em módulos ES6. Separação de responsabilidades em arquivos menores (<150 linhas cada). Centralização dos event listeners em navigation.js.

Dependências Circulares nos Imports

JavaScript
Resolvido

Problema

Após a modularização, alguns módulos importavam uns aos outros criando ciclos. Isso causava erros de "Cannot access before initialization".

Causa

Alguns módulos importavam uns aos outros criando ciclos de dependência que o sistema não conseguia resolver.

Solução

Criação de um módulo state.js independente sem imports. Uso de injeção de dependência onde necessário. Reorganização da hierarquia de módulos.

Porta 3000 Sempre Ocupada

Server
Resolvido

Problema

Ao reiniciar o servidor, frequentemente aparecia o erro "EADDRINUSE: address already in use :::3000". Processos Node.js órfãos ficavam rodando em background.

Causa

Crashes do servidor ou fechamento abrupto do terminal deixavam processos Node.js ativos. O novo servidor não conseguia bindar na mesma porta.

Solução

Script de restart que mata processos existentes: Get-Process node | Stop-Process -Force. Graceful shutdown implementado no servidor com SIGTERM handler.

Dark Mode Não Aplicava em Todos Componentes

CSS
Resolvido

Problema

Ao ativar o dark mode, alguns elementos (modais, cards, inputs) continuavam com cores claras. A experiência visual ficava inconsistente.

Causa

CSS estava usando cores hardcoded (#ffffff, #f5f5f5) em vez de variáveis CSS. O seletor [data-theme="dark"] não sobrescrevia essas cores.

Solução

Migração completa para CSS Variables. Definição de todas as cores em :root e [data-theme="dark"]. Criação de variáveis semânticas como --surface, --text.

Módulos ES6 Não Carregavam

Server
Resolvido

Problema

Console mostrava erro: "Failed to load module script... MIME type must be a JavaScript MIME type". Os imports no browser não funcionavam.

Causa

Express estava servindo arquivos .js com Content-Type incorreto ou faltando. Browsers modernos exigem application/javascript para módulos ES6.

Solução

Configuração correta do express.static com MIME types. Verificação que todos os imports usam extensão .js. Adição de type="module" no script tag.

Arquivos com Mais de 150 Linhas

Refactoring
Resolvido

Problema

Requisito do projeto: nenhum arquivo deveria ter mais de 150 linhas para manter a manutenibilidade. Vários arquivos excediam esse limite.

Causa

Código legado com múltiplas responsabilidades por arquivo, excedendo o limite de 150 linhas estabelecido.

Solução

Divisão em 32 módulos JS e 24 módulos CSS. Cada módulo com responsabilidade única seguindo o limite de 150 linhas.

Estado Desincronizado Entre Módulos

JavaScript
Resolvido

Problema

Após operações CRUD, a interface não atualizava automaticamente em todos os componentes afetados.

Causa

Cada módulo mantinha sua própria cópia do estado sem comunicação entre eles.

Solução

Criação de state.js centralizado com sistema de eventos customizados para notificar mudanças.
